# MacSense — 프로젝트 정리 문서

## 한 줄 요약
macOS 메뉴바 상주 앱. 사용자의 반복적인 마우스 클릭/메뉴 사용 패턴을 백그라운드에서 감지하고, AI가 더 빠른 단축키나 워크플로우를 자연어로 추천해주는 서비스.

## 이름
**MacSense** (MakeSense 언어유희 + Mac 타겟)

## 수업 조건
- AI를 활용한 서비스 (웹, 어플, 프로그램)
- 5분 아이디어 발표 → 동료 투표로 팀 구성 (3~4명)
- 투표로 채택 시 추가 점수

---

## 핵심 컨셉
**"기존 단축키 앱은 사용자가 이미 단축키를 알아야 한다. MacSense는 AI가 먼저 발견한다."**

- 사용자가 비효율적인 행동을 하고 있는지 인식할 필요 없음
- AI가 반복 패턴을 감지해서 먼저 알려줌
- 단축키 추천 + 워크플로우 제안 + 주간 리포트

---

## 작동 흐름

```
[이벤트 리스너] → [로그 저장] → [반복 감지] → [DB/LLM 매칭] → [팝업 추천]
   항상 대기       클릭마다       N번 반복 시     로컬+AI        알림 1줄
```

### 상세 흐름
1. **CGEventTap**으로 마우스 클릭 이벤트 감지 (이벤트 리스너, 폴링 아님)
2. 클릭 좌표 (x, y) 획득
3. **AXUIElementCopyElementAtPosition(systemWide, x, y)** → 클릭한 UI 요소 반환
4. 해당 요소의 Role(버튼/메뉴), Title(이름) 읽기
5. 부모 요소 타고 올라가며 경로 구성 → "Finder > Menu:File > MenuItem:New Folder"
6. 로그 저장: `{시각, 앱, 경로}` → 로컬 SQLite
7. 같은 행동 하루 N번(예: 3번) 이상 반복 감지 → 단축키 DB 매칭 or LLM 질의
8. macOS 알림 팝업: "이 작업을 오늘 5번 하셨어요. ⌘⇧N으로 가능합니다"

---

## 기술 스택

| 구성 | 기술 |
|---|---|
| 앱 형태 | macOS 메뉴바 앱 (RunCat처럼 NSStatusBar 상주) |
| 언어 | Swift (macOS 네이티브) |
| UI 이벤트 캡처 | Accessibility API (AXUIElement) + CGEventTap |
| 패턴 분석 | N-gram 빈도 분석 (실시간) + LLM (배치 분석) |
| 단축키 DB | 앱별 JSON (초기 5~8개 앱) |
| 자연어 추천 | LLM API (Claude/GPT) |
| 데이터 저장 | 로컬 SQLite (서버 없음) |
| 알림 | macOS 네이티브 알림 |

## AI 역할

### AI가 하는 것
1. **단축키 DB에 없는 행동의 자동화 레시피 제안** — 공식 단축키가 없는 복합 행동에 대해 없는 단축키를 지어내지 않고, Shortcuts/Automator로 만들 수 있는 사용자 자동화 방법을 제안.
2. **복합 시퀀스 패턴 인식** — 하루치 로그를 LLM에 전달하여 앱을 넘나드는 워크플로우 패턴 발견.
3. **자연어 코칭** — "File > Save As 대신 ⌘⇧S 한 번이면 됩니다" 같은 자연어 설명 생성.
4. **주간 리포트 생성** — 습득한 단축키, 절약 시간, AI 인사이트를 종합한 리포트.

### AI 없이도 되는 것 (규칙 기반)
- 단순 빈도 카운팅 (같은 메뉴 3번 클릭 감지)
- 단축키 DB 직접 매칭
- 통계 집계

### AI 비중에 대한 판단
- 수업 조건 "AI를 활용한 서비스"는 LLM API 활용으로 충분히 만족
- 핵심 차별점: "고유 데이터(Accessibility API 행동 로그) + LLM"의 조합
- 단순 ChatGPT 래퍼가 아니라 사용자 고유 행동 데이터를 LLM에 넘기는 구조

## 패턴 분석 방법

### 방법 1: N-gram 빈도 분석 (실시간용, 구현 간단)
- 연속된 N개 이벤트를 묶어서 빈도 카운트
- "File_Open → NewFolder → Rename" 3-gram이 3번 반복 → 패턴 감지
- 장점: 구현 10줄. 단점: N을 미리 정해야 함

### 방법 2: LLM에 로그 전달 (배치 분석용, 가장 현실적)
- 하루 끝에 전체 로그를 LLM에 넘기고 "반복 패턴 찾아줘" 요청
- 패턴 인식 + 단축키 매칭 + 자연어 설명이 한 번에 해결
- 장점: 직접 알고리즘 구현 불필요. 단점: API 비용, 실시간 불가

### 현실적 조합
- **실시간**: N-gram으로 단순 빈도 카운팅 → 즉시 팝업
- **하루 끝**: 전체 로그를 LLM에 전달 → 데일리/주간 리포트

## Accessibility API 상세

### AXUIElement로 알 수 있는 것
- `kAXRoleAttribute`: 요소 유형 (AXButton, AXMenuItem, AXTextField 등)
- `kAXTitleAttribute`: 요소 이름 ("New Folder", "Save As" 등)
- `kAXDescriptionAttribute`: 보조 설명 (툴바 SF Symbol 버튼 등 title이 빈 경우 fallback)
- `kAXHelpAttribute`: 툴팁 텍스트
- `kAXParentAttribute`: 부모 요소 (메뉴 경로 추적용)
- `kAXChildrenAttribute`: 자식 요소 목록 (컨테이너 내부 라벨 추출용)
- `kAXPositionAttribute`: 화면 좌표

### 권한
- 시스템 설정 → 개인정보 보호 → 접근성에서 앱 허용 필요
- 입력 모니터링 권한도 별도 필요 (CGEventTap용)
- `AXIsProcessTrusted()`로 권한 확인 가능

### 검증 결과 (PoC 완료)

CGEventTap + AX 조합으로 실제 클릭 이벤트 추적 검증 완료. 다음 정보가 안정적으로 추출됨:

| 앱 카테고리 | 추적 가능 항목 | MVP 적합성 |
|---|---|---|
| Finder | 파일/폴더/메뉴 작업 | ✅ 메인 |
| Chrome | 탭/창/북마크/페이지 작업 | ✅ 메인 |
| Notes | 메모/폴더/서식 작업 | ✅ 메인 |
| 기타 네이티브 앱 | 메뉴/툴바/파일/윈도우 일부 | ⚠️ 확장 후보 |
| Electron 앱 | 파일명/다이얼로그/버튼 일부 | ⚠️ 베타 |
| Dock | 앱 아이템 전부 | ✅ |
| 게임/풀스크린 앱 | 미지원 | ❌ |

**핵심 발견**
- title이 비어 있어도 `description` → `help` → 자식 요소 순으로 fallback하면 거의 모든 빈 케이스가 채워짐
- 카카오톡까지 완벽하게 추적되어 한국 사용자 타겟에 강점
- Electron 앱은 콘텐츠 영역(에디터, 캔버스)은 한계지만, 시맨틱 컨트롤(다이얼로그/버튼/메뉴)은 일부 추적 가능

## 서비스 바운더리

### 하는 것
- macOS 메뉴바 상주, 백그라운드 이벤트 감지
- 반복 행동 감지 → 단축키/워크플로우 추천 팝업
- 추천 이력 기록 → 습득 여부 추적
- 주간 리포트 생성 (습득한 단축키, 절약 시간, AI 인사이트)

### 안 하는 것
- 타이핑 내용(텍스트) 수집 ❌
- 비밀번호 필드 감지 시 자동 중단
- 자동 매크로 실행 ❌ (보안 리스크)
- 키 리매핑 ❌ (기존 도구 영역)
- 서버 전송 ❌ (모든 데이터 로컬)

### 초기 지원 앱 (MVP)
- **Finder**: 파일/폴더 작업 대표, 메인 데모
- **Chrome**: 웹 브라우징 작업 대표
- **Notes**: 기본 메모/문서 작업 대표
